> **이 문서의 두 가지 역할**
> 1. 우측 패널 패턴의 **배포용 지식 단위**(machine-readable + 교육용)

## 1. 한 줄 정의

> 메인 영상(주로 실시간 방송) 시청을 **끊지 않은 채**, 화면 우측 일부에 떠서 관련 기능·옵션을 **머물며 탐색**하게 하는 패턴.

---

## 2. 판단 — 언제 이 패턴을 쓰는가

### 결정 규칙 (단일 질문)
> **"메인 콘텐츠 시청 유지"와 "기능 묶음 탐색"이 *둘 다* 필요한가?** → 예 = 우측 패널.**

둘 중 하나만 필요하면 다른 패턴이다. 아래 분기표로 판정한다.

| 상황 | 시청 유지 | 기능 탐색 | → 권장 패턴 |
|---|:---:|:---:|---|
| 시청 중 채널/옵션/관련 채널 접근 | ✅ | ✅ | **우측 패널** |
| 채널 번호·재생 정보 잠깐 확인 후 사라짐 | ✅ | ❌(머물지 않음) | 가이드 레이어(OSD) |
| 시청 떠나 대량 콘텐츠 둘러보기 | ❌ | ✅ | 그리드 뷰(Grid) |
| 콘텐츠 발견·감정 몰입 | ❌ | ❌ | 파노라마(Panorama) |
| 설정처럼 좌(카테고리)–우(상세) 동시 노출 | ❌ | ✅ | 스플릿 뷰(Split) |

### 헷갈리는 경계: 우측 패널 vs OSD
- **OSD**: 보고 *사라진다*. 시선이 머물지 않음. (예: 채널 번호 바)
- **우측 패널**: 시선이 *머물며 탐색*한다. 기능 묶음에 포커스가 이동함.
- 판정: "사용자가 그 위에서 ↑/↓로 항목을 고르는가?" → 그렇다면 우측 패널.

### 안티패턴 (이건 우측 패널이 아니다)
- 패널이 화면 절반 이상을 덮어 영상이 가려진다 → 시청 유지 실패. **Split/Full로 재분류.**

## 3. 동작 — 리모컨 내비게이션 스펙 🔒

> TV에서 패턴의 절반은 "포커스가 어떻게 움직이는가"다. hover가 없으므로 **focus가 왕.**

### 시선·포커스 흐름 (역 L자)
시청 중(화면 중앙) → `→` 호출 시 우측 상단으로 포커스 진입 → 우측 벽 따라 하강.
**→ 가장 중요한 기능을 패널 최상단에 둔다.**

### 키맵

| 상황 | 키 | 동작 |
|---|---|---|
| 진입 | `→` 또는 `옵션/OK` | 우측 패널 호출, 포커스 = **최상단 첫 항목** |
| 항목 이동 | `↑` / `↓` | 패널 내 항목 간 이동 |
| 하위 진입 | `→` 또는 `OK` | 해당 항목의 2depth(상세) 진입 |
| 시청 복귀 | `←` | 패널 닫고 영상으로 즉시 복귀 |
| 닫기 | `뒤로` | 현 단계 닫기 (2depth → 1depth → 패널 종료) |

### 핵심 규칙
- **래핑 금지**: 최하단에서 `↓` 입력 시 순환하지 않고 **멈춘다**. (시니어의 "끝을 모르겠다" 혼란 방지 — §4 게이트와 연결)
- **자동 닫힘**: 무동작 타임아웃은 *충분히 길게* 두거나 설정으로 끄게 한다. 짧은 자동 닫힘은 시니어에게 "사라졌다" 불안을 유발한다.
- **시청 방해 최소화**: 패널은 우측 일부만 점유, 메인 영상은 계속 재생(딤·축소 최소).
- **깊이 제한**: 패널에서 들어가는 depth ≤ 2.
- **되돌리기 보장**: 모든 동작은 `뒤로`로 안전하게 취소·복귀 가능(실수해도 안심).

## 4. 게이트 — 시니어 통과 기준 🔒 (Pass / Fail)

> 이 게이트가 자산의 **자동 검수 기준**이 된다. 하나라도 Fail이면 표준 패턴으로 승인하지 않는다.
> 수치는 **U+tv 펀더멘털 가이드(`U_tv_Foundation_Guide.md`)** 기준으로 채웠다. 파운데이션에 없는 값은 ⚠️로 표시.

| # | 기준 | 통과 조건 | 측정 방법 |
|---|---|---|---|
| G1 | 글자 가독 | 패널 항목 라벨 **= 24pt** (파운데이션 §3.4 · 2~3m 기준 — 확정) | Pretendard 크기 토큰 준수 검사 |
| G2 | 명도 대비 | 텍스트·UI 대비 **7.1:1 이상** — 반투명 영상 위에서도 동일 (파운데이션 §6.1) | 대비비 자동 계산 |
| G3 | 포커스 가시성 | 현재 포커스 항목이 한눈에 구분 (**Soft White `#F5F5F5` 고대비 면 반전**, §8.1①) | focus 상태 토큰 적용 검사 |
| G4 | 항목 수 | 첫 진입 시 보이는 항목 **≤ 6** (권장 4~5 — 확정) | 스크롤 없이 보이는 항목 수 카운트 |
| G5 | 용어 난이도 | 외래어·약어 금지, 쉬운 한국어 | 라벨 사전 대조 (예: "PIP"✗ → "작은 화면으로 보기"✅, "EPG"✗ → "편성표"✅) |
| G6 | 음성 안내 | 모든 항목이 TTS로 읽히는 라벨 보유, 포커스 시 음성 안내 가능 | 라벨 음성 메타 존재 검사 |
| G7 | 깊이 제한 | 패널 기점 depth ≤ 2 | 내비 트리 깊이 측정 |
| G8 | 되돌리기 | 모든 동작 `뒤로`로 복귀 가능 | 동작별 역경로 존재 검사 |

## 5. 재료 — UDS-TV 토큰 / 컴포넌트 매핑 ✏️

> 값은 **U+tv 펀더멘털 가이드(`U_tv_Foundation_Guide.md`)**에서 가져온다. 파운데이션에 없는 항목은 ⚠️로 표시하고 §맨 아래에 모아 둔다.

### 컴포넌트
- `RightPanel/Container` — 우측 점유, 영상 비파괴
- `RightPanel/ListItem` (focusable) — 항목 단위
- `RightPanel/Header`, `RightPanel/Divider`

### 상태 토큰 (focus가 핵심)
| 상태 | 값 | 비고 |
|---|---|---|
| 기본(비포커스) | 텍스트 `Gray700 #CBCBCD` | 포커스 항목과 명도 차이를 둠 |
| 포커스 | **면 반전**: 면 `Soft White #F5F5F5` + 글자 `Soft Black #111` | **필수** — G3 직결. 면 반전(§8.1①). Primary 분홍(#FA057F) ❌ · 포커스는 중립 고대비 (파운데이션 §2.1·2.2) |
| 선택(확정) | **Primary `#FA057F` 체크/도트 마커** + 라벨 중립 유지(`Soft White`) | 확정 — Radio 선택 규칙과 일관. 마커만 Primary(액션·선택값), 행 전체는 중립. 포커스(면 반전)와 비충돌 — 포커스 시 마커는 어둡게 반전 |
| 비활성 | 텍스트 `Gray500 #6D6D72` (필요 시 opacity 디밍은 §8.3 최소대비 보장) | |

### 레이아웃 토큰
- 항목 간격: `--space-250` (20px) (파운데이션 §4.1)
- 항목 글자: **Pretendard SemiBold 24pt** (파운데이션 §3.4 — 핵심 내비 텍스트)
- 패널 폭: **480px** (1920 기준 화면 25%, 영상 비파괴)
- 항목 높이: **96px 고정** (라벨 24pt + 상하 패딩 `--space-400` 32px). 포커스 시 면 반전만, **크기 불변**(§8.2)

### ✅ 우측 패널 고유 결정 — 확정 (2026-06-22, 파운데이션 기준)
파운데이션이 답하지 않던 패널 고유 값을 파운데이션 원칙에 맞춰 확정. (G1·G4 검증 가능)

| # | 결정 | 확정값 | 근거 |
|---|---|---|---|
| 1 | 항목 글자 크기 | **24pt** | §3.4(2~3m 시니어 본문 최소 24pt)가 §3.3 라벨 18pt보다 우선. 패널 항목 = 핵심 내비 텍스트 |
| 2 | 첫 화면 항목 수 | **≤ 6** (권장 4~5) | 시니어 "끝을 모르겠다" 방지(§3 래핑 금지와 연결), 6단계 단순 구조 원칙 정합 |
| 3 | 선택(확정) 표시 | **Primary 마커 + 라벨 중립** | Radio 선택 규칙 일관, 포커스(면 반전)와 비충돌 |
| 4 | 패널 폭 | **480px** | 1920 기준 25%, 영상 비파괴(§1·§2 안티패턴) |
| 5 | 항목 높이 | **96px 고정** | 포커스 타깃 충분, §8.2 크기 불변 |

> 색·글자·간격 출처: `U_tv_Foundation_Guide.md` (v1.0)

## 6. 예시 ✏️

### ✅ Good
실시간 시청 중 `→` → "옵션" 패널 호출 → 최상단 **"관련 채널"** 포커스 → 영상은 계속 재생 → `←`로 즉시 복귀. 라벨은 "편성표", "작은 화면으로 보기" 등 쉬운 한국어.

### ❌ Bad
- 패널이 화면 절반을 덮어 영상이 가려짐 → 시청 유지 실패 (안티패턴, §2).
- 라벨에 "EPG", "VOD", "PIP" 약어 사용 → G5 Fail.
- 최하단에서 `↓` 무한 순환 → 래핑 금지 위반(§3).
- 자동 닫힘이 너무 빨라 시니어가 "사라졌다" → §3 위반.

## 7. 머신리더블 화면 조립 블록

> 빌더가 소비하는 형식. 모바일팀 `uds-rule-Patterns.md` 구조의 TV판.

```
[Pattern] RightPanel
[Trigger] watching=live  &  key=Right|Option
[Surface] right-overlay, width=480px, video-non-blocking=true
[Entry] focus = item[0]
[Nav]
  Up/Down   -> move within items
  Right/OK  -> enter detail (depth<=2)
  Left      -> close panel, return to video
  Back      -> step back one level
[Rule] wrap=false; auto-dismiss=long|off; visible-items<=6
[Compose]
  [Header]  align=left
  [List]    items (focusable, height=96px, label=24pt-semibold) {
              label(easy-korean), tts-meta,
              focus: invert(face=soft-white, text=soft-black),
              selected: primary-marker + neutral-label
            }
[Gate] G1..G8 must pass
```

## 7-B. 화면 스키마 (생성기 레퍼런스) ★ LLM few-shot

> 입력 예: *"실시간 시청 중 우측 옵션 패널. 관련 채널(포커스)·편성표·작은 화면으로 보기·음성 다중(선택)·화면 비율."*
> `layout:"rightPanel"` = 본 영상 유지 + 우측 패널. `list` 항목은 focusable(포커스=면 반전, 선택=Primary 마커).

```json
{ "screen": {
  "name": "실시간 옵션",
  "size": [1920, 1080], "bg": "core/soft-black", "layout": "rightPanel",
  "children": [
    { "type": "text", "style": "font-title-7-700", "color": "core/soft-white", "content": "옵션" },
    { "type": "list", "items": [
      { "label": "관련 채널", "focus": true },
      { "label": "편성표" },
      { "label": "작은 화면으로 보기" },
      { "label": "음성 다중", "selected": true },
      { "label": "화면 비율" }
    ]}
  ]
}}
```

> ※ 첫 진입 시 보이는 항목 ≤6(§4 G4). 라벨은 쉬운 한국어(약어 금지, §G5). list item = `RightPanel/ListItem`로 조립.

## 8. 이양 테스트 (Transfer Test) — Phase 1 종료 조건

> "전파됐다"의 판정 기준. **우리 팀이 개입하지 않은 상태**에서 검증한다.

1. 기획자/개발자에게 이 문서 **한 장만** 전달.
2. "실시간 시청 중 옵션 패널 화면을 만들어 달라" 요청.
3. 산출물이 **게이트 G1~G8을 모두 통과**하는가?
   - 통과 → 이 패턴 "자산화 완료".
   - 실패 → 실패한 게이트 번호 = 카탈로그가 덜 명시적인 지점. 그 섹션을 보완하고 재시험.
