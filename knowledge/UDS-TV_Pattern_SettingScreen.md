**기본 3원칙 (요청 반영)**
1. **브래드크럼은 우상단** — 현재 위치 경로 표시.
2. **최상단이 아니면 좌측에 Back 큐** (76px 칼럼, chevron-back). 최상단이면 생략.
3. **콘텐츠는 메뉴별 블럭(모듈)으로 구성** — 블럭을 세로로 쌓는다.

## 1-B. 화면 스키마 (생성기 레퍼런스) ★ LLM few-shot

> 웹 생성기는 이 JSON 형식으로 출력한다. `layout:"settingScreen"` + 블록 `lnb`(items/selected)·`breadcrumb`(items)·`card`(children). 카드 안에 제목/설명 text + 컨트롤(Toggle/Radio/Dropdown).
> 입력 예: *"왼쪽에 LNB가 있는 설정에서 자동 전원 끄기 설정화면. 전원 자동 끄기 토글(켜짐) + 끄는 시간(사용 안 함/1~4시간, 2시간 선택)."*

```json
{ "screen": {
  "name": "자동 전원 끄기 설정",
  "size": [1920, 1080],
  "bg": "core/soft-black",
  "layout": "settingScreen",
  "children": [
    { "type": "lnb", "items": [
      { "label": "화면 설정" },
      { "label": "소리 설정" },
      { "label": "자동 전원 끄기 설정", "selected": true },
      { "label": "시간 설정" },
      { "label": "시스템 정보" }
    ]},
    { "type": "breadcrumb", "items": ["홈", "설정", "일반", "자동 전원 끄기 설정"] },
    { "type": "card", "children": [
      { "type": "group", "direction": "horizontal", "align": "spaceBetween", "children": [
        { "type": "text", "style": "font-title-7-700", "color": "core/soft-white", "content": "전원 자동 끄기" },
        { "type": "component", "component": "Toggle", "props": { "State": "On" } }
      ]},
      { "type": "text", "style": "font-label-4-300", "color": "core/gray-600", "content": "TV를 켜 둔 채 설정한 시간이 지나면 자동으로 전원을 꺼요." }
    ]},
    { "type": "card", "children": [
      { "type": "text", "style": "font-title-7-700", "color": "core/soft-white", "content": "자동으로 끄는 시간" },
      { "type": "text", "style": "font-label-4-300", "color": "core/gray-600", "content": "전원을 끌 시간을 선택해 주세요." },
      { "type": "group", "direction": "horizontal", "gap": "space-400", "children": [
        { "type": "component", "component": "Radio", "props": { "State": "Default", "label": "사용 안 함" } },
        { "type": "component", "component": "Radio", "props": { "State": "Default", "label": "1시간" } },
        { "type": "component", "component": "Radio", "props": { "State": "Selected", "label": "2시간" } },
        { "type": "component", "component": "Radio", "props": { "State": "Default", "label": "3시간" } },
        { "type": "component", "component": "Radio", "props": { "State": "Default", "label": "4시간" } }
      ]}
    ]}
  ]
}}
```

- **토글 카드**: `card` → `group(horizontal, spaceBetween)`[제목 text · `Toggle`] + 설명 text.
- **라디오 카드**: `card` → 제목 text + 설명 text + `group(horizontal, gap=space-400)`[`Radio` ×N, 하나 Selected].
- **드롭다운 행**: `group(horizontal, spaceBetween)`[라벨 text · `Dropdown`].
- **푸터(적용형)**: 맨 끝에 `group(horizontal)`[`Button`(취소)·`Button`(적용)]. 즉시반영형은 생략.
- LNB·breadcrumb의 **마지막/selected 항목 = 현재 화면 제목과 일치**.

#### V5. 버튼 + 우측 현재값 표시 (상세)
- **버튼** = 하위 지면으로 설정하러 진입(310px, V3와 동일 버튼). **우측 텍스트** = 현재 적용값 요약(Soft White 50%, 24px).
- 예: "15세 관람가까지 시청 가능" / "숨김" / "모든 앱 실행 가능" / "잠금 채널 15개".
- **복합값(수량/한도형)**: `라벨 | 현재/최대` + progress bar.
  - 예 일일 시청 한도 "매일 | 2시간 30분 / 5시간 00분" (현재 50% / 최대 40%) + 진행바(Soft White 20% 트랙 + 채움, 채움 ~59%).
- ⚠️ **우측 현재값 50% ≈ 5:1 디밍** — 현재 설정 상태는 중요 정보인데 7.1 미달(시스템 이슈 C-2, §8).

- 재사용 컴포넌트: 라디오 = `UDS-TV_Component_Radio.md`, 버튼 = `UDS-TV_Component_Button.md`.
- 라디오는 2지든 N지든 **가로 나열 + gap 32**가 공통. 포커스/선택은 라디오 컴포넌트 규칙 그대로(선택=Primary 도트, 포커스=면 반전).

## 3-C. 설정 화면 2모드 (Mode A / Mode B) ★ 분류 기준

설정 항목의 **상호작용 방식**으로 화면을 두 모드로 나눈다. 같은 골격(§1) 위에서 모듈만 다르다.

| 모드 | 항목 형태 | 적용/취소 | 예 | 대표 변형 |
|---|---|---|---|---|
| **Mode A 허브형** | **버튼 → 진입** + 우측 현재값 | 하위 지면에서 설정·확정 | 자녀 보호 모드 `235:70956` | **V5** (복합/하위 지면 있는 설정) |
| **Mode B 직접설정형** | **인라인 라디오** | **선택 즉시 적용** (진입·취소/적용 없음) | 시청각 보조 방송 `235:60739` | V1·V2 (인라인 라디오) |

### Mode B 직접설정형 구조 (시청각 보조 방송 `235:60739`)

- **섹션 그룹 헤더**: **40px Bold** 섹션 타이틀("시각 보조 설정" / "청각 보조 설정") + 그 아래 카드(여러 행). **섹션 간 gap 60**.
- **행** = 타이틀(메인 36px + 서브 설명 / 하위 속성 28px) + 라디오 그룹.
- **라디오 레이아웃 3변형** (옵션 수에 따라):

| 레이아웃 | 옵션 수 | 간격 | 예 |
|---|---|---|---|
| 가로 1줄 | 2~3개 | gap 32 | 꺼짐/켜짐 · 하단/상단/좌측 · 좁음/보통/넓음 |
| 2행 그리드 | 6~8개 | 컬럼 gap 32 / 행 gap 8 | 음성안내 속도 6개(2×3) · 글자색/배경색 8개(2×4) |
| 가로 5개 | 5개 | gap 32 | 자막 크기 아주작게~아주크게 |

- **라이브 프리뷰**: 자막 크기 설정에 **720×388 미리보기 박스**(테두리 Soft White 50% 4px)로 실제 자막 **WYSIWYG** 미리보기. → **V4(이미지카드 프리뷰)의 실시간판**.

> 참고: 자막 프리뷰 안의 **퓨어블랙은 "실제 자막 렌더"라 OK**(콘텐츠 영역) — UI 표면 퓨어블랙 금지(§0·§8)와 별개.
> ⚠️ **미선택 라디오 라벨 50% ≈ 5:1 디밍** — 접근성(시청각 보조) 페이지인데 옵션이 흐린 아이러니(시스템 이슈 C-2, §8).

### ⚠️ 브래드크럼 대비 (최우선)
상위 경로 **2.0:1**, 현재 항목 **4.9:1** — 2~3m·시니어 환경에서 "내가 어디 있는지"를 못 읽는다. 길찾기(wayfinding) 정보라 더 중요.
- 조치: 상위 경로 ≥ Gray700(또는 Soft White 70%↑), 현재 항목 = Soft White 100%로 상향. 디밍으로 위계 주되 **최소 대비선 아래로 내리지 말 것**.

### ⚠️ 서브텍스트 디밍
모듈 서브텍스트 60%(6.2:1)도 7.1 미달. 라디오·LNB·탭과 동일한 불투명도 디밍 패턴 — **설정 전반 공통 규칙으로 "위계는 색/굵기, 디밍 최소 대비 보장"** 정리 필요.

### ⚠️ 디밍 미달 추가 항목 (V5·Mode B — 시스템 이슈 C-2)
- **V5 우측 현재값 50%(~5:1)** — 현재 설정 상태는 중요 정보인데 미달.
- **Mode B 미선택 라디오 라벨 50%(~5:1)** — 접근성 페이지인데 옵션이 흐림.
- 위 둘 다 서브텍스트와 같은 불투명도 디밍 계열 → 공통 규칙으로 일괄 해결 대상.

### ✅ 자막 라이브 프리뷰 내 퓨어블랙은 예외
Mode B 자막 크기 프리뷰(720×388) 안의 퓨어블랙은 **실제 자막 렌더(콘텐츠)**라 OK. UI 표면 퓨어블랙 금지(§0)와 별개 — 콘텐츠 영역에는 적용 안 함.
