# UDS-TV 화면 생성기 — 실행 방법 (디자이너용)

문장 입력 → 우리 컴포넌트로 화면 생성. **3가지 사용 방법**이 있고, 손이 덜 가는 순서야.

---

## 방법 1. 나한테 바로 말하기 (설치 0, 지금 가능)
대화창에 "○○ 설정 페이지 만들어줘" 라고 하면 내가 피그마에 바로 만들어 줌.
- 장점: 설치·서버 아무것도 필요 없음. 피그마에 진짜 프레임으로 들어감.
- 지금까지 만든 화면들이 다 이 방식. **시안 단계엔 이게 제일 빠름.**

---

## 방법 2. 웹앱 (브라우저에서 입력 → 미리보기)
`uds-tv-plugin/web/` + `backend/` 가 그거야. 문장 넣으면 브라우저에서 화면을 **미리보기**하고 JSON을 뽑아줌.

> 결과를 "피그마에 바로" 넣진 못함(그건 플러그인 영역). 웹앱은 **빠른 미리보기 + JSON 출력**까지. JSON을 방법 3 플러그인 ②에 붙이면 피그마에 들어감.

**한 번만 세팅** (Mac/PC + 인터넷 필요. 개발자 없어도 따라할 수 있음):
1. **Node.js** 설치 → https://nodejs.org (LTS 버전).
2. 터미널 열고:
   ```
   cd "<이 폴더 경로>/uds-tv-plugin/backend"
   npm install
   ANTHROPIC_API_KEY=sk-여기에_키 npm start
   ```
   - Anthropic API 키는 https://console.anthropic.com 에서 발급.
   - `UDS-TV backend on :8787` 가 뜨면 성공.
3. 브라우저에서 **http://localhost:8787** 열기 → 문장 입력 → "화면 생성".

> 이 대화창에서 바로 명령을 실행해보고 싶으면, 입력칸에 `!` 를 앞에 붙여 명령을 치면 돼 (예: `! cd ... && npm start`). 출력이 여기로 들어와.

---

## 방법 3. Figma 플러그인 (피그마에 직접 생성)
`uds-tv-plugin/manifest.json` 을 피그마에 등록하면, 피그마 안에서 화면을 직접 만들어.

1. Figma 데스크톱 → **Plugins → Development → Import plugin from manifest…** → `manifest.json` 선택.
2. 컴포넌트가 있는 파일에서 플러그인 실행.
3. 사용:
   - **① 자유문장**: 방법 2의 백엔드 주소(`http://localhost:8787/generate`)를 넣고 문장 입력 → 피그마에 생성.
   - **② JSON 직접**: 웹앱에서 복사한 스키마 JSON을 붙여넣기 → 백엔드 없이 바로 생성.

---

## 한눈에 비교
| | 설치 | 결과 위치 | 추천 |
|---|---|---|---|
| 1. 나한테 말하기 | 없음 | 피그마 프레임 | 시안·실무 (지금) |
| 2. 웹앱 | Node+키 1회 | 브라우저 미리보기 + JSON | 여러 시안 빠르게 보기 |
| 3. 플러그인 | import 1회 (+① 쓰려면 서버) | 피그마 프레임 | 팀 배포 |

---

## 방법 4. 웹에 배포 (Vercel — URL만 열면 누구나)
서버를 클라우드에 한 번 올리면, 팀원이 로컬 설치 없이 **URL 접속**만으로 웹앱을 씀.

1. 이 저장소를 GitHub에 올림(또는 Vercel CLI).
2. [vercel.com](https://vercel.com) → New Project → 이 저장소 import.
   - **Root Directory = `uds-tv-plugin`** 로 지정(여기에 `vercel.json`·`package.json`·`web/`·`api/` 있음).
3. **Environment Variables**에 `ANTHROPIC_API_KEY` 추가 → Deploy.
4. 배포된 URL 열면 끝. (웹앱=`/`, API=`/generate`·`/lint`, 함수=`api/`)

> 서버리스 함수(`api/generate.js`)는 로컬 서버와 **같은 `lib/generate.js`를 재사용**(코드 중복 없음). 저장소 .md/.json이 함수에 안 잡히면 임베드 어휘로 자동 폴백.

## 서버 기능 요약 (생성 파이프라인)
```
spec → [LLM] 스키마 → autoFix(색·용어·중립대비) → validateVocab(카탈로그 밖 사용 차단)
     → runGates(G1/G2/§8.3/G4/G5/G8) → (문제 남으면) LLM 재생성(최대 2회) → { screen, _meta }
```
- `POST /generate {spec}` → `{ screen, _meta:{fixes,warns,vocabErrors,attempts} }`
- `POST /lint {screen}` → 검수만(LLM 없이)
- `GET /health` → 상태·모델·키 유무

## 폴더 구조
```
uds-tv-plugin/
├─ manifest.json · code.js · ui.html      ← Figma 플러그인 (방법 3)
├─ web/index.html                          ← 웹앱 (방법 2·4)
├─ api/generate.js · api/lint.js           ← Vercel 서버리스 함수 (방법 4)
├─ vercel.json · package.json              ← 배포 설정/의존성 (방법 4)
└─ backend/
   ├─ server.js                            ← 로컬 서버 + 웹앱 서빙 (방법 2)
   ├─ lib/generate.js                      ← 생성 핵심(LLM 루프·어휘검증) ★공유
   ├─ gates.js                             ← 게이트/자동교정 ★공유(플러그인과 동일 기준)
   └─ package.json
```
어휘·문법(루트): `uds-tv-screen-catalog.json` · `uds-tv-glossary.json` · `UDS-TV_ScreenSchema.md`

> 핵심: LLM은 "자연어→화면 스키마(JSON)"만. 조립·검수·교정·어휘검증은 결정적 코드가. 로컬·서버리스·플러그인이 **같은 `gates.js`/`generate.js`** 를 공유해 기준이 어긋나지 않음.
